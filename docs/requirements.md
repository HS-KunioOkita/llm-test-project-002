
# システム名
中小卸売業向け受発注管理システム

# 目的
営業担当、受注担当、管理者が利用し、受注登録、承認、出荷管理、売上集計を行うWebシステムを構築する。

# スコープ

- 取引先管理
- 商品管理
- 受注登録
- 受注承認
- 出荷ステータス管理
- CSV入出力
- 日次売上集計バッチ
- 管理画面
- REST API

# ユーザー種別

- 営業担当
- 受注担当
- 管理者
- 閲覧専用ユーザー

# 機能要件

- 取引先を登録・更新・検索できること
- 商品を登録・更新・検索できること
- 営業担当は受注を登録できること
- 受注担当または管理者のみ受注承認できること
- 承認済み受注のみ出荷可能であること
- 出荷ステータスは「未出荷 / 出荷済み / キャンセル」とする
- 取引先・商品はCSVでインポートできること
- 日次で売上集計バッチを実行できること
- すべての更新系処理で監査ログを残すこと
- 受注一覧は条件検索・ページングに対応すること

# 非機能要件

- 同時利用者 100 名程度
- 一般的な操作は 2 秒以内
- テストコードを含めること
- Docker Compose でローカル起動できること
- README にセットアップと実行手順を書くこと

# 技術制約

- Frontend: React + TypeScript
- Backend: Node.js + TypeScript
- DB: PostgreSQL
- API: REST
- Test: Jest or Vitest
- ORM利用可
- 外部SaaS利用禁止
